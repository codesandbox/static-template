Take Home Activity - Sidebar

  • Instructions can be found here

    • User Story
    A sidebar is one of the most indispensable components of any dashboard UI. Not only does it help a user change the context of the app easily it helps the developer break down the apps into separate sections
  • In this take home you will be implementing a static sidebar with plain HTML and CSS
      • NOTE - All the absolute units should be converted to relative (rem) units, except in case of borders. 1rem = 16px by default.
    • Body
      1. The font used is Space Grotesk which comes from Google Fonts
      2. The font is used document wide, but should it fail to render in any case, the fallback fonts should be rendered.
    • Container
      1. The sidebar should have a border on three sides i.e. top, right, and bottom. There is NO border on the left
      2. The border should be rounded on three sides (mentioned above) ONLY! Which means the top-left and bottom-left corner should not be rounded.
      3. The border color is of 1px, solid style, and color #dadada
      4. The sidebar heading should be transformed to uppercase.
      5. There should be a margin on 1rem
      6. A padding of 2rem should be applied in Y direction (top & bottom), No padding in the X direction (left & right)
      7. A width of 10rem and a height = 80% of the viewport height.
    • Heading
      1. Should be colored blueviolet.
      2. A padding of 0.5rem in the X direction. No padding in Y direction.
      3. Size of the font should be 12px
      4. The text should be converted to all UPPERCASE with a spacing of 2px in between
    • Item
      1. A padding of 0.5 rem all around the content.
      2. Each item should be separated by a distance of 4px.
      3. On hover the font should turn semi-bold (with weight of 600) and the background color to #f0f0f0
      4. The cursor should turn into a pointer when the mouse is on this item.

    Solution

      3. Use the following styles
        • Heading should be indigo in color.
        • The paragraph should be gray in color.

      2. An image of width and height 100 with alt text as the member’s name. This shall serve as the avatar of the person.
      3. An h2 heading to display the name
      4. An h3 heading to display the role
      5. A paragraph text to display the bio.
